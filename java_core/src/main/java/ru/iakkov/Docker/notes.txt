One Server Problems

-Два приложения хотят занят один и тот же сетевой порт
-Два приложения используют один и тот же файл, вследствие чего одно из них может заблокировать его (file block)
-Два приложения хотят использовать две разные версии одной и той же глобальной библиотеки
-Одно приложение изменяет переменную окружения, которую использует другое приложение. Из-за чего ломается одно из приложений
-Два приложения конкурируют за одни и те же системные ресурсы (CPU, RAM, SSD).

Виртуализация - это технология, которая обеспечивает изоляцию вычислительных процессов, выполняемых на одном физическом устройстве,
и разделение ресурсов между этими процессами (CPU, RAM, SSD).

На сервер устанавливается host операционная система (hypervision) которая умеет запускать несколько других guest ОС (virtual machine)
и выделяет каждой из них свой набор ресурсов.
Каждая ВМ требует установки для нее операционной системы, что влечет за собой:
1 - Накладные расходы на установку, обслуживание, обновление ОС
2 - Доп. потребление CPU, RAM, SSD
3 - Покупка лицензий для каждой установленной ОС
